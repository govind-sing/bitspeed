
# Bitespeed Backend Task: Identity Reconciliation

This service provides an **Identity Resolution** engine designed to consolidate multiple customer contact points (emails and phone numbers) into a single, unified "Primary" identity.

**Live Endpoint:** [https://bitspeed-1vrp.onrender.com/identify/](https://bitspeed-1vrp.onrender.com/identify/)

---

## 🚀 The Challenge

The core problem involves linking different orders made with different contact information to the same person. The system handles:

* **New Customer Creation:** Initializing a "Primary" contact.
* **Secondary Linking:** Adding new contact info (new email/phone) to an existing customer.
* **Identity Merging:** Bridging two previously separate contact "islands" when a new request provides information belonging to both.

---

## 🛠️ Tech Stack

* **Runtime:** Node.js (ES Modules)
* **Language:** TypeScript
* **Framework:** Express.js
* **Database:** PostgreSQL (Hosted on Neon.tech)
* **ORM:** Prisma
* **Deployment:** Render

---

## 🧠 Design Philosophy

To ensure high performance and data integrity, the reconciliation logic follows a **Root-Tracer** approach:

1. **Discovery:** On every request, the system fetches all contacts matching the provided email or phone.
2. **Primary Identification:** It traces back to the oldest "Primary" record in the cluster.
3. **Merge Logic:** If a request links two different Primary records, the newer Primary is downgraded to "Secondary," and all its descendants are re-linked to the older Root Primary.
4. **Consolidation:** The final response is generated by a recursive-style fetch of the entire family tree, ensuring the **Primary contact's information is always at index `[0]**` of the arrays as per requirements.

## ⚙️ Local Setup

1. **Clone the repository:**
```bash
git clone <your-repo-link>
cd bitespeed-identity

```


2. **Install dependencies:**
```bash
npm install

```


3. **Set up Environment Variables:**
Create a `.env` file and add your PostgreSQL connection string:
```env
DATABASE_URL="postgresql://user:password@host:port/neondb?sslmode=require"

```


4. **Run Database Migrations:**
```bash
npx prisma migrate dev

```


5. **Start Development Server:**
```bash
npm run dev

```

## 🧪 API Usage

**Endpoint:** `POST /identify`

**Request Body:**

```json
{
  "email": "mcfly@hillvalley.edu",
  "phoneNumber": "123456"
}

```

**Response Body:**

```json
{
  "contact": {
    "primaryContatctId": 1,
    "emails": ["lorraine@hillvalley.edu", "mcfly@hillvalley.edu"],
    "phoneNumbers": ["123456"],
    "secondaryContactIds": [23]
  }
}

```
